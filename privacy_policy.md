## 3) Categories of Personal Data We Process

Depending on how you use ArcWealth, we may process:

### 3.1 Account and identity data

- User identifier (internal ID);
- email address;
- first name and last name;
- username;
- date of birth;
- role (e.g., user/student/teacher/admin);
- email verification status and timestamps.

### 3.2 Student profile and educational context data

- school name;
- country and city;
- grade level;
- learning goal;
- guardian email (for under-18 signups);
- optional avatar image URL;
- optional gender field (including "Prefer not to say").

### 3.3 Learning and engagement data

- lesson/topic progress and completion status;
- XP events and XP totals;
- rank, level, streak metrics;
- achievements earned;
- textbook progress;
- onboarding tutorial state.

### 3.4 Classroom/social data

- teacher-student links;
- classroom messages/announcements and assignments;
- teacher nudges/congratulations;
- friend requests and friend links;
- leaderboard/public profile display fields (for eligible users).

### 3.5 Authentication and security data

- password hash (never plaintext password);
- session IDs and session timestamps;
- OAuth account linkage data (provider and provider account identifier);
- one-time token hashes for email verification/password reset;
- 2FA state (including encrypted TOTP secret and hashed recovery codes when enabled);
- rate-limiting keys (including IP-derived values in memory).

### 3.6 Technical/client-side data

- strictly necessary session/security cookies;
- short-lived 2FA-related cookies;
- limited local storage entries used for in-app learning UX.

### 3.7 Communications data

- transactional email metadata and content needed to send:
  - verification emails,
  - password reset emails,
  - guardian registration notices,
  - teacher-triggered classroom notifications.

## 5) Why We Process Data and Legal Bases (GDPR Art. 6)

We process personal data for the purposes below.

### 5.1 Service delivery and account operations

- Create/manage accounts, authenticate users, maintain sessions, deliver app features.
- **Legal basis:** Art. 6(1)(b) GDPR (performance of contract).

### 5.2 Security and abuse prevention

- Prevent unauthorized access, enforce security controls, manage 2FA, rate-limit suspicious behavior.
- **Legal basis:** Art. 6(1)(f) GDPR (legitimate interests in security and service integrity); in some contexts Art. 6(1)(b).

### 5.3 Educational functionality

- Provide lessons, track progress, show achievements/rankings, enable teacher-student workflows.
- **Legal basis:** Art. 6(1)(b) GDPR; and where school contracts apply, potentially Art. 6(1)(e) or Art. 6(1)(f) depending on institutional framework.

### 5.4 Communications

- Send verification, password reset, and service notifications.
- **Legal basis:** Art. 6(1)(b) GDPR (necessary service communications) and Art. 6(1)(f) for security/operational notices.

### 5.5 Legal compliance

- Comply with legal obligations, regulatory requests, and lawful enforcement actions.
- **Legal basis:** Art. 6(1)(c) GDPR.

### 5.6 Consent-based processing (if introduced)

- Where we rely on consent (e.g., optional non-essential tracking in future), users can withdraw consent at any time.
- **Legal basis:** Art. 6(1)(a) GDPR.

## 7) Cookies and Similar Technologies

We use limited cookies/local storage to operate the Services.

### 7.1 Cookies we currently use for core functionality

- `arcwealth_session` (HTTP-only session cookie; authentication);
- `arcwealth_2fa_pending` (short-lived security cookie for 2FA challenge state);
- `arcwealth_totp_enroll` (short-lived security cookie for TOTP enrollment state).

These are necessary for authentication and account security.

### 7.2 Local storage currently used

- `reviewQueue` for in-app learning review queue state;
- a landing page theme preference key (`arcwealth-landing-theme`) for UI preference.

If we add non-essential analytics/marketing cookies, we will update this policy and, where required, implement consent controls before activating those trackers.

## 10) Data Retention

We keep personal data only as long as needed for the purposes described above or as required by law.

Current retention logic includes:

- active account/profile data: retained while account remains active;
- authentication/session records: retained for security and account operation, then deleted/expired according to lifecycle needs;
- verification/reset tokens: short-lived and invalidated on use or expiry;
- 2FA security artifacts: retained while enabled or until revoked/reset;
- educational progress records: retained while account is active, unless deletion is requested and legally permissible;
- logs/operational metadata: retained for security, troubleshooting, and legal compliance for reasonable periods.

Where feasible, we delete or anonymize data after retention periods end.
